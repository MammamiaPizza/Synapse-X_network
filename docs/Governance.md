# Governance & Human-in-the-Loop (HITL)
**กลุ่ม 18 — S-XAN Network Project**

---

## 1. Governance — การกำกับดูแลและตรวจสอบ

เพื่อให้เครือข่าย S-XAN มีความน่าเชื่อถือและตรวจสอบได้ เราได้วางโครงสร้างการบริหารจัดการดังนี้

### Audit Trails — ระบบบันทึกเหตุการณ์
ทุกการเข้าถึงระบบใน **Session Layer** จะถูกบันทึก (Logs) อย่างละเอียดเพื่อตรวจสอบย้อนกลับได้ในกรณีที่ระบบทำงานผิดปกติ โดยข้อมูลทั้งหมดจะถูก **เข้ารหัสขั้นสูงสุด** เพื่อความปลอดภัยของผู้ใช้

### Medical Oversight — การกำกับดูแลโดยผู้เชี่ยวชาญ
การปรับเปลี่ยนค่าพารามิเตอร์สำคัญ เช่น ระดับความแรงสูงสุดของสัญญาณ จะต้องได้รับ **อนุมัติจากผู้เชี่ยวชาญทางการแพทย์** ผ่านอินเทอร์เฟซใน **Application Layer** เสมอ

### Incident Management — มาตรฐานการจัดการข้อผิดพลาด
ระบบมีเกณฑ์การตัดสินใจที่ชัดเจน — หากเครือข่ายมีความหน่วงเกิน **200 ms** ระบบจะหยุดการส่งข้อมูลทันทีเพื่อป้องกันความสับสนของสมองผู้ใช้

## 2. Human-in-the-Loop (HITL) — มนุษย์เป็นผู้ตัดสินใจสูงสุด

> **หัวใจของสถาปัตยกรรม:** เทคโนโลยีต้องอยู่ภายใต้การควบคุมของมนุษย์ **100%**

### Physiological Kill-switch — ระบบตัดการทำงานทันที
ผู้ใช้งานสามารถสั่ง **ตัดการเชื่อมต่อเครือข่าย** ได้ทันทีในระดับกายภาพ (Manual Override) ผ่านพฤติกรรมที่กำหนดไว้ล่วงหน้า เช่น:
- การกะพริบตาถี่ๆ
- การกัดฟันในรูปแบบเฉพาะ

ระบบจะรับรู้สัญญาณเหล่านี้ทันทีผ่านเซ็นเซอร์ใน **Physical Layer**

### User Consent Mechanism — สิทธิในการยืนยัน
ก่อนเริ่ม Augmentation หรือการเชื่อมต่ออวัยวะเทียม ระบบจะต้องรอ **สัญญาณยืนยัน (Handshake)** จากเจตจำนงของผู้ใช้ (User Intent) เสมอ — **ไม่มีการเชื่อมต่ออัตโนมัติโดยที่ผู้ใช้ไม่รู้ตัว**

### Fail-safe Mode — ระบบป้องกันความเสี่ยงอัตโนมัติ
หากระบบตรวจพบว่าผู้ใช้มีความเครียดของระบบประสาทสูงเกินไป (Neural Stress) S-XAN จะลดระดับสัญญาณลงเหลือเพียง **Safe Mode** โดยอัตโนมัติ เพื่อรอให้มนุษย์เป็นผู้ตัดสินใจว่าจะรันระบบต่อหรือหยุดทำงาน
